Theory

Issue:: "why not just parse the C++ source text to find function calls?" The answer is 
that C++ is extremely hard to parse correctly. Templates, macros, overloading, and inheritance 
make text-based analysis fragile and incomplete.

LLVM:LLVM solves this by first compiling your code into a simplified, universal format called LLVM IR 
(Intermediate Representation). This IR is much simpler than C++ source — no macros, no 
templates, everything is explicit. The pass then analyzes this IR instead of raw source code.

Original C++: int add(int a, int b) { return a + b; }
define i32 @add(i32 %a, i32 %b) 
{
entry:
 %result = add i32 %a, %b
 ret i32 %result
}

It can also detect direct and undirect calls. Solve pointer analysis.

Clang: Clang is a modern compiler front-end designed for the C language family, including C, C++, and Objective-C. It is known for its fast compilation times, 
user-friendly error messages, and modular architecture. Clang translates source code into LLVM Intermediate Representation (IR), which is a platform-independent code format that can be optimized and compiled into machine code by LLVM.
